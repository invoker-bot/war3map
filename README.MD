## Overview
War3map is a module to convert between a Object representation of WarCraft III (.w3x) data and `war3map` format files.

![TranslationExample](docs/img/example.png)

War3map is written in `TypeScript`, offering type safety and improved development experience.

Here's some places where you can use this library:
* You need to decompile a map of Warcraft 3. 
* You want to write a map editor for Warcraft 3.
* You want to optimize or encrypt your Warcraft 3 map.

`UnitsObject` supports both RoC `war3mapUnits.doo` v7/subversion 9 and TFT
v8/subversion 11 layouts, including byte-preserving duplicate item drops.
`EnvironmentObject` supports terrain `war3map.w3e` v11 and Reforged v12
texture flag layouts.

### Trigger files

|   Type | Object → War  | War → Object  | File |
|   ---  |   :-:   |   :-:   |   ---    |
| LUA    | ![check][ok] | ![check][ok] | war3map.lua  |
| JASS   | ![check][ok] | ![check][ok] | war3map.j    |
| Strings  | ![check][ok] | ![check][ok] | war3map.wts  |
| Custom Text Triggers | ![check][ok] | ![check][ok] | war3map.wct |
| Trigger Definitions | ![check][ok] | ![check][ok] | war3map.wtg |

`TriggerNamesObject` preserves full `war3map.wtg` bytes and exposes Reforged
root metadata, type counts, variable definitions, and tree-object summaries.
Pass `triggerData` from `UI\TriggerData.txt` to parse GUI functions and
parameter lists; without it, the raw payload is still preserved.


### Map files

| Type | Object → War  | War → Object  | File   |
|   ---    |    :-: |   :-: |   --- |
| Info File    | ![check][ok] | ![check][ok] | war3map.w3i  |
| Imported Files  | ![check][ok] | ![check][ok] | war3map.imp  |
| Pathing      | ![check][ok] | ![check][ok] | war3map.wpm  |
| Shadow map    | ![check][ok] | ![check][ok] | war3map.shd |
| Menu Minimap | ![check][ok] | ![check][ok] | war3map.mmp |
| Game Configuration | ![check][ok] | ![check][ok] | *.wgc |

`SoundsObject` exposes pitch and distance values as numeric fields and keeps the
World Editor metadata/tail markers structured for byte-for-byte round trips.
`GameConfigurationObject` supports the `.wgc` files generated by World Editor AI
test runs.

### Image and payload-preserving files
These files expose validated headers or editable text while preserving payload
bytes for byte-for-byte archive rebuilds.

* ![check][ok] BLP1/BLP2 minimap and imported images (`war3mapMap.blp`, `*.blp`)
* ![check][ok] TGA and DDS preview, pathing, and imported images (`*.tga`, `*.dds`)
* ![check][ok] Image conversion helpers for PNG, TGA, DDS, BLP1 JPEG/paletted images, and BLP2 BGRA/DXT images
* ![check][ok] MDX model chunks, materials, geosets, animation tracks, geoset animations, texture animations, global sequences, pivots, bones, helpers, attachments, cameras, events, collision shapes, ribbon emitters, and particle emitters 2 (`*.mdx`)
* ![check][ok] WAV/MP3 audio headers, including empty WAV override placeholders (`*.wav`, `*.mp3`)
* ![check][ok] WAI AI editor headers plus condition/function/string summaries (`*.wai`)
* ![check][ok] Text resource pass-through for SLK/FDF/TOC/INI/TXT/AI files
* ![check][ok] Opaque fallback payloads through `RawFileObject`

MDX files also expose `knownChunkErrors` when a known chunk is preserved but
cannot be safely decoded into the structured view. Treat these diagnostics as a
conversion warning and keep the original chunk bytes unless the format variant is
understood.

## Getting Started

### Prerequisites

* [NODEJS](https://nodejs.org/)
**VERSION >= 10** 

### Installation

The easiest and fastest way is to use it as a library. Just simply type the followings:
* npm
```sh
npm install war3map
```

### Import

Create your `.ts` or `.js` file, and type as follows:
```ts
import * as war3map from "war3map";
```

_To get more examples or usages for war3map, see [Documentation](https://invoker-bot.github.io/war3map)._

### Archive Support

`MapArchiveObject` can read and write `.w3m`, `.w3x`, and `.mpq` archives through
[node-stormlib](https://github.com/invoker-bot/node-stormlib). `node-stormlib`
only handles the MPQ container; this package parses the map files inside the
archive. Install `node-stormlib` or set `NODE_STORMLIB_PATH` to a local checkout
such as `D:\Projects\GitHub\node-storm`.

```ts
import { MapArchiveObject } from "war3map";

const map = new MapArchiveObject();
map.readArchive("example.w3x");
map.writeArchive("out/example.w3x", { overwrite: true });
```

### Development

To get a local copy up and running follow these simple steps:

1. Clone the repo
```sh
git clone https://github.com/invoker-bot/war3map.git
```
2. Enter the correct directory
```sh
cd war3map
```
3. Install NPM packages
```sh
npm install
```
4. Run All Tests
```
npm test
```
